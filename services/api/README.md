# api

主后端目录，负责应用编排、接口输出、数据持久化和与回测引擎协作。

## 建议子目录

```text
app/
  api/                    HTTP 接口层
  application/            用例编排与流程控制
  domain/                 领域对象与业务规则
  infrastructure/         数据库、外部数据、日志等基础设施
  schemas/                请求响应模型与内部 DTO
```

## 目录职责

- `app/api`: 提供接口入口，不直接写复杂业务。
- `app/application`: 串联“保存策略、发起回测、保存结论、加入手册”等流程。
- `app/domain`: 承载核心业务语义与规则约束。
- `app/infrastructure`: 提供数据库、配置、日志、数据访问能力。
- `app/schemas`: 定义 API 输入输出结构与模块间传输结构。
