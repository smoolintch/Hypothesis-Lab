# integration

集成测试目录，用于验证模块之间的协作是否正确。

## 建议子目录

```text
api/                      接口与应用编排相关集成测试
backtest/                 回测链路与结果持久化相关集成测试
```

## 当前已落地基线

1. `test_repo_guardrails.py`：验证仓库最小护栏基线仍然成立。
2. `api/test_api_smoke.py`：验证 `services/api` 最小 FastAPI 应用可创建，且默认文档端点可访问。

说明：
1. 这批测试只覆盖工程底座可验证性，不覆盖任何业务接口。
2. 当前不依赖数据库启动，不要求本地 PostgreSQL 已就绪。

## 当前阶段建议覆盖

1. 策略保存后可发起回测
2. 回测结果可落库并重新查询
3. 结论可绑定到正确的策略或回测记录
4. 交易手册条目可正确引用结果与结论
