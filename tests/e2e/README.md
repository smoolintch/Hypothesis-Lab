# e2e

端到端测试目录，用于验证真实用户闭环是否成立。

## 建议子目录

```text
core-flow/                核心主流程测试
```

## 当前已落地基线

1. 已保留 `core-flow/` 目录作为后续主链路 E2E 的唯一入口。
2. 当前阶段只建立目录和说明，不强行引入完整 Playwright 套件。
3. 仓库当前可验证重点仍以护栏检查和 API smoke test 为主。

## MVP 必测主链路

1. 新建策略假设卡
2. 保存策略
3. 发起回测
4. 查看结果
5. 保存结论
6. 加入交易手册

如果这条链路无法稳定通过，MVP 就不算真正完成。
