# 验收标准与测试清单

## 1. 目标
本文件用于定义每个阶段的验收标准和测试重点，确保开发不是“代码写完”而是“能力可验收”。

## 2. P0 核心验收
1. 用户可以创建并保存策略假设卡。
2. 用户可以发起回测。
3. 系统可以返回稳定的回测结果。
4. 用户可以看懂核心指标与曲线。
5. 用户可以保存结论。
6. 用户可以加入交易手册。
7. 关键数据刷新后不丢失。

## 3. 测试分层

### 3.1 单元测试
1. 规则模板校验
2. 指标计算
3. 回测逻辑核心函数

### 3.2 集成测试
1. 策略保存后可以发起回测
2. 回测结果可正确入库
3. 结论可绑定到正确结果

### 3.3 E2E 测试
1. 新建策略
2. 发起回测
3. 查看结果
4. 保存结论
5. 加入交易手册

## 4. 当前阶段验收重点

### 阶段 0
1. 文档齐备
2. 契约已冻结
3. 工程初始化可启动

### 阶段 1
1. 策略表单可保存
2. 字段校验合理
3. 编辑逻辑可回填

### 阶段 2
1. 回测可运行
2. 同一输入结果一致
3. 错误可见可定位

### 阶段 3
1. 结果页可读
2. 结论可保存
3. 手册链路闭环成立
