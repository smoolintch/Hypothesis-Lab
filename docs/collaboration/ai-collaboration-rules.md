# AI 协作规则

## 1. 目标
本文件用于约束 AI 在本项目中的协作方式，确保多轮协作下仍保持边界清晰、实现稳定、文档同步。

`AGENTS.md` 是项目级执行入口，本文件用于说明协作原则；真正开工前应优先阅读 `AGENTS.md`。

## 2. 核心规则
1. 任何实现都必须服从 MVP 闭环优先级。
2. 不得主动引入 Out of Scope 功能。
3. 跨模块改动前，先检查并更新 `contracts/` 与 `architecture/`。
4. 新增技术路线或重大取舍时，必须补一条 ADR。
5. 修改接口时，必须同步更新接口契约与相关测试。
6. 修改领域模型时，必须同步更新数据模型文档。
7. 优先做最小可交付版本，不跳过中间闭环。

## 3. 协作流程建议
1. 先确认目标模块和边界。
2. 先读取 `docs/project/current-status.md` 与 `docs/project/session-handoff.md`。
3. 先更新文档，再动跨边界代码。
4. 先实现主链路，再补体验细节。
5. 完成后补测试与验收项。

## 4. 不建议的协作方式
1. 一次同时改动前端、后端、回测引擎且不更新契约。
2. 在没有冻结接口时直接并行开发。
3. 为了“更高级”提前引入微服务、复杂基础设施或过度抽象。
