# 会话交接

## 1. 当前状态
当前没有处于中断中的开发任务。

如果未来某次开发被中途打断、上下文丢失、API 枯竭，或需要换一个 Agent 接手，应立即把本文件改写为下面的格式。

## 2. 交接模板

### 当前目标
正在完成的唯一主目标是什么。

### 已完成
本轮已经做完了哪些事情。

### 已修改文件
列出本轮已改动的文件。

### 当前停在哪里
说明目前准确停在什么步骤，为什么还没完成。

### 下一步最安全动作
给出一个明确、可直接执行的下一步，不要写模糊建议。

### 验证情况
1. 已验证什么
2. 没验证什么
3. 为什么没验证

### 风险与注意事项
说明接手者继续开发时最容易踩坑的地方。

### 接手前必读
列出必须先读的文档和文件。

## 3. 使用规则
1. 任何未完成任务离场前必须更新本文件。
2. 若任务已完成，应把本文件恢复为“当前没有处于中断中的开发任务”。
3. 不允许让接手者从聊天记录里猜上下文。
