# 任务拆解与开发顺序

## 1. 目标
本文件用于把 `MVP开发计划.md` 中的阶段目标拆成实际执行顺序，避免开发过程中交叉打架。

## 2. 当前建议顺序

### 阶段 0：开工前冻结
1. 冻结技术方案与技术约束
2. 冻结系统架构与模块边界
3. 冻结领域模型
4. 冻结主链路接口契约
5. 冻结验收标准
6. 建立 AI 协作规则

### 阶段 1：工程底座
1. 初始化 `apps/web`
2. 初始化 `services/api`
3. 初始化基础测试框架
4. 初始化环境变量和配置规范

### 阶段 2：策略输入链路
1. 实现规则模板 schema
2. 实现策略卡表单
3. 实现策略卡保存与编辑

### 阶段 3：回测链路
1. 接入标准化样本数据
2. 实现回测引擎最小能力
3. 实现结果持久化

### 阶段 4：结果与沉淀链路
1. 实现结果页
2. 实现模板化摘要
3. 实现结论保存
4. 实现加入交易手册

### 阶段 5：补全可用性
1. 策略列表
2. 复制策略卡
3. 最近实验记录
4. 历史回测记录

## 3. 并行开发前提
只有下列文档冻结后，前后端才建议并行：

1. `contracts/domain-model.md`
2. `contracts/api-contracts.md`
3. `product/user-flow-and-page-states.md`
