# 任务拆解与开发顺序

## 1. 目标
本文件用于把 `MVP开发计划.md` 中的阶段目标拆成实际执行顺序，避免开发过程中交叉打架。

## 2. 当前建议顺序

### 阶段 0：文档冻结与工程底座

#### 0A. 开工前冻结
1. 冻结技术方案与技术约束
2. 冻结系统架构与模块边界
3. 冻结领域模型
4. 冻结主链路接口契约
5. 冻结规则模板 schema v1
6. 冻结页面主流程与关键状态
7. 冻结验收标准
8. 建立 AI 协作规则

#### 0B. 工程底座
1. 初始化 Git 仓库并连接远程仓库
2. 确定前端工作区与包管理方案
3. 确定 Python 包管理与迁移工具方案
4. 建立环境变量规范与 `.env.example`
5. 建立最小测试基线
6. 建立最小 CI 骨架
7. 准备固定样例策略和固定行情数据
8. 初始化 `apps/web`
9. 初始化 `services/api`

### 阶段 1：策略输入链路
1. 实现规则模板 schema 对应的前端表单渲染
2. 实现策略卡表单
3. 实现策略卡保存与编辑
4. 打通保存后再次加载与回填

### 阶段 2：回测链路
1. 接入标准化样本数据
2. 实现回测引擎最小能力
3. 实现 `StrategySnapshot -> BacktestRun -> BacktestResult`
4. 实现结果持久化

### 阶段 3：结果与沉淀链路
1. 实现结果页
2. 实现模板化摘要
3. 实现结论保存
4. 实现加入交易手册

### 阶段 4：补全可用性
1. 策略列表
2. 复制策略卡
3. 最近实验记录
4. 历史回测记录

## 3. 并行开发前提
只有下列文档冻结后，前后端才建议并行：

1. `contracts/domain-model.md`
2. `contracts/api-contracts.md`
3. `contracts/rule-template-schema-v1.md`
4. `product/user-flow-and-page-states.md`

## 4. 当前推荐执行顺序
1. 先完成 `0A. 开工前冻结`
2. 再完成 `0B. 工程底座`
3. 再进入 `阶段 1：策略输入链路`
