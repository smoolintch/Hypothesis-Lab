# 用户流程与页面状态

## 1. 目标
本文件用于定义 MVP 主流程、页面跳转条件、关键状态和异常流，确保前端实现不会只覆盖 happy path。

## 2. MVP 主流程
```text
首页
  -> 新建策略假设
    -> 保存策略卡
      -> 发起回测
        -> 查看结果
          -> 填写结论
            -> 加入交易手册
```

## 3. 路由建议
| 页面 | 路由 | 作用 |
| --- | --- | --- |
| 首页 / 控制台 | `/` | 策略列表入口、最近实验、快速开始 |
| 新建策略页 | `/strategy-cards/new` | 创建策略卡 |
| 编辑策略页 | `/strategy-cards/{id}/edit` | 编辑已保存策略卡 |
| 回测结果页 | `/backtests/{run_id}` | 轮询回测状态并展示结果 |
| 交易手册页 | `/handbook` | 查看已纳入手册的条目 |

## 4. 页面跳转规则
1. 首页点击“新建策略假设”进入 `/strategy-cards/new`。
2. 新建策略首次保存成功后，页面跳转为 `/strategy-cards/{id}/edit`。
3. 编辑页点击“发起回测”时，系统必须先做前端校验，再执行保存，再创建回测运行。
4. 回测创建成功后，立即跳转 `/backtests/{run_id}`，不在编辑页原地等待。
5. 结果页保存结论成功后，留在当前页并更新结论区域状态。
6. 结果页加入手册成功后，留在当前页并展示“已加入手册”，同时提供跳转 `/handbook` 入口。
7. 交易手册页点击条目时，应能回到对应策略或结果详情。

## 5. 主流程状态机
| 步骤 | 进入条件 | 成功结果 | 失败结果 |
| --- | --- | --- | --- |
| 新建策略 | 用户从首页进入新建页 | 表单可编辑 | 页面初始化失败 |
| 保存策略 | 表单校验通过并提交 | 获得 `strategy_card_id` | 表单错误或保存失败 |
| 发起回测 | 当前策略卡已通过校验 | 获得 `run_id` 并跳转结果页 | 保存失败或回测创建失败 |
| 轮询回测 | 进入 `/backtests/{run_id}` | 状态变为 `succeeded` | 状态变为 `failed` 或查询失败 |
| 查看结果 | 成功拿到 `BacktestResult` | 展示指标、曲线、交易明细 | 结果缺失或格式异常 |
| 保存结论 | 用户填写结论并提交 | 结论进入已保存状态 | 结论校验失败或保存失败 |
| 加入手册 | 结论满足加入手册条件 | 条目创建成功 | 条件不满足或写入失败 |

## 6. 通用页面状态
每个页面至少覆盖以下五类状态：
1. 初始状态
2. 加载中
3. 空状态
4. 成功状态
5. 错误状态

另外：
1. 含提交动作的页面必须覆盖“提交中”状态。
2. 含异步结果的页面必须覆盖“轮询中 / 等待中”状态。
3. 含表单的页面必须覆盖“字段级校验错误”状态。

## 7. 页面级状态定义

### 7.1 首页 `/`
| 状态 | 触发条件 | UI 要求 | 主要动作 |
| --- | --- | --- | --- |
| 初始加载 | 首次进入页面 | 展示骨架屏或 loading | 无 |
| 无策略空状态 | `strategy_card_count = 0` | 强调主 CTA | “新建策略假设” |
| 有数据状态 | 存在策略卡或最近实验 | 展示最近策略、最近回测 | 新建、继续编辑、查看结果 |
| 加载失败 | 首页概览接口失败 | 展示错误提示和重试 | “重新加载” |

建议文案：
1. 空状态标题：`还没有任何策略假设`
2. 空状态说明：`先创建第一张策略卡，开始一次完整验证闭环。`

### 7.2 策略假设编辑页 `/strategy-cards/new`、`/strategy-cards/{id}/edit`
| 状态 | 触发条件 | UI 要求 | 主要动作 |
| --- | --- | --- | --- |
| 新建模式 | 进入 `/new` | 加载默认值 | 保存、发起回测 |
| 编辑加载中 | 进入 `/edit` 且尚未拿到详情 | 表单骨架屏 | 无 |
| 编辑模式 | 成功拿到策略详情 | 表单回填 | 保存、发起回测 |
| 字段校验失败 | 提交前校验未通过 | 高亮字段错误与顶部提示 | 修正后重提 |
| 保存中 | 点击保存后等待接口返回 | 按钮禁用，显示 loading | 等待完成 |
| 保存成功 | 保存接口成功 | 轻提示，不跳页 | 继续编辑或发起回测 |
| 发起回测中 | 点击“发起回测”后 | 按钮禁用，阻止重复提交 | 跳转结果页 |
| 保存失败 | 保存接口失败 | 展示错误提示 | 重试 |

交互约束：
1. `发起回测` 按钮的行为等价于“校验 + 保存 + 创建回测”。
2. 用户离开编辑页且存在未保存内容时，必须弹出离开确认。
3. 当规则模板切换时，旧模板参数必须清空，避免脏参数混入提交。

### 7.3 回测结果页 `/backtests/{run_id}`
| 状态 | 触发条件 | UI 要求 | 主要动作 |
| --- | --- | --- | --- |
| 运行状态查询中 | 初次进入页面 | 显示 run loading | 无 |
| 回测排队中 | `status = queued` | 展示等待说明和轮询提示 | 返回编辑页 |
| 回测运行中 | `status = running` | 展示进度占位和轮询提示 | 返回编辑页 |
| 结果加载中 | `status = succeeded` 但结果接口未完成 | 展示结果骨架屏 | 等待 |
| 结果成功 | 成功拿到结果 | 展示指标、曲线、交易明细、摘要 | 填写结论、加入手册 |
| 无结果 | `status = succeeded` 但结果为空 | 展示异常提示 | 重新查询或返回 |
| 回测失败 | `status = failed` | 展示标准错误信息 | 返回编辑页、重新发起回测 |
| 结论未填写 | 已有结果但无结论 | 展示结论表单 | 保存结论 |
| 结论已保存 | 已存在结论 | 展示只读或可编辑视图 | 更新结论 |
| 已加入手册 | 已存在手册条目 | 展示成功标识 | 前往手册页 |

轮询规则：
1. 进入页面立即查询一次 `GET /api/backtests/{run_id}`。
2. 当状态为 `queued` 或 `running` 时，每 `3` 秒轮询一次。
3. 当状态变为 `succeeded` 后停止状态轮询并请求结果接口。
4. 当状态变为 `failed` 后停止轮询。

### 7.4 交易手册页 `/handbook`
| 状态 | 触发条件 | UI 要求 | 主要动作 |
| --- | --- | --- | --- |
| 加载中 | 首次进入页面 | 列表骨架屏 | 无 |
| 手册为空 | `total = 0` | 展示空状态说明 | 返回首页或创建策略 |
| 已有条目 | 存在手册条目 | 展示列表 | 查看详情 |
| 加载失败 | 接口失败 | 展示错误提示 | 重新加载 |
| 移出确认 | 未来支持移出时 | 弹确认框 | 确认 / 取消 |

## 8. 表单与交互边界
1. 用户不能在未保存的脏表单上直接查看“最新结果”；必须先保存或放弃修改。
2. 回测运行中禁止再次点击“发起回测”。
3. 当 `Conclusion.next_action != add_to_handbook` 时，“加入交易手册”按钮应禁用或给出原因说明。
4. 当结果页刷新后，页面必须能根据 `run_id` 自恢复。
5. 当编辑页通过不存在的 `strategy_card_id` 进入时，展示 `404` 风格错误态，而不是空白页。

## 9. 页面级反馈文案建议
1. 保存成功：`策略已保存`
2. 发起回测中：`正在创建回测任务...`
3. 回测排队中：`回测已提交，正在等待执行`
4. 回测运行中：`回测执行中，请稍候`
5. 回测失败：`回测失败，请检查策略参数或稍后重试`
6. 结论保存成功：`结论已保存`
7. 加入手册成功：`已加入交易手册`

## 10. 本阶段不处理的交互
1. 多标签页编辑冲突提示
2. 手册条目复杂筛选
3. 策略卡拖拽排序
4. 实时 WebSocket 回测进度推送
