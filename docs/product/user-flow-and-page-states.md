# 用户流程与页面状态

## 1. 目标
本文件用于定义 MVP 主流程、关键页面和页面状态，确保前端实现不会遗漏空状态、错误状态和异步状态。

## 2. MVP 主流程

```text
首页
  -> 新建策略假设
    -> 保存策略卡
      -> 发起回测
        -> 查看结果
          -> 填写结论
            -> 加入交易手册
```

## 3. 核心页面
1. 首页 / 控制台
2. 策略假设编辑页
3. 回测结果页
4. 交易手册页

## 4. 通用页面状态
每个页面都应至少考虑：

1. 初始状态
2. 加载中
3. 空状态
4. 成功状态
5. 错误状态

## 5. 页面级状态建议

### 5.1 首页
1. 无策略空状态
2. 有最近实验状态
3. 数据加载失败状态

### 5.2 策略假设编辑页
1. 新建模式
2. 编辑模式
3. 表单校验失败状态
4. 保存成功状态
5. 发起回测中状态

### 5.3 回测结果页
1. 回测运行中状态
2. 结果成功返回状态
3. 无结果状态
4. 回测失败状态
5. 结论未填写状态
6. 已加入手册状态

### 5.4 交易手册页
1. 手册为空状态
2. 已有手册条目状态
3. 移出手册确认状态
